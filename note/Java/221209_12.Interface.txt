221209 [12] 인터페이스 (Interface)

1. 인터페이스의 이해
    
	(1) 작업지시서:
   	 - 실제 구현된 것이 전혀 없는 기본 설계도
  	 - 객체를 생성할 수 없고, 클래스 작성에 도움을 줄 목적으로 사용됨
   	 - 미리 정해진 규칙에 맞게 구현하도록 표준을 제시하는 데 사용됨
  	 - 추상메소드와 상수만을 멤버로 가질 수 있음
	(2) 다형성을 가능하게 함 (하나의 객체를 다양한 type으로 만들 수 있음)
	(3) 객체의 부속품화 -> 다양한 객체를 제품의 부속품처럼 개발자 마음대로 변경 가능
 	(4) 실제 개발에 적용시키기가 쉽지 않기 때문에 고민이 필요함


2. 인터페이스 문법
	(1) 'class' 대신 'interface' 예약어를 사용함
  	(2) 실제 구현된 기능 없이 추상메소드와 상수만이 존재
	(3) private 불가 (상수가 메소드를 만들 때 private 접근제한자는 불가함)
