221208 [11] 추상클래스 (Abstract Class)

1. 추상클래스:
    “추상적으로 정의(혹은 선언)할 테니, 객체를 사용할 사용자가 꼭 override 하여 사용하세요!”


2. 추상클래스의 문법
	a. abstract (추상클래스 및 추상메소드를 선언하는 예약어) 이용
	b. 추상클래스에는 하나 이상의 추상메소드가 포함됨. 추상메소드는 정의만 하고 구현은 안 함.
  	c. 추상메소드에는 메소드의 선언부만 있고, 실행부(구현부/Body)는 없음
  	d. 추상클래스에서는 메소드 선언만 하고, 구현은 상속 받는 클래스에서 함.
      추상클래스에서 정의된 기능은 하위 클래스에서 상세 구현.
  	e. 클래스의 프레임만 구성. 직접 객체 생성 불가능.


3. 추상클래스 상속과 일반클래스 상속의 차이
    부모클래스의 메소드를 자식클래스에서 변경할 일이 없다면, 일반클래스를 이용한 상속 사용.
    자식클래스에서 꼭 재정의 해야 하는 부분이 있다면, 추상클래스를 이용한 상속 사용.


4. final (제한자): 추상클래스와 정반대되는 개념.
    
	(1) 클래스 앞에 붙일 경우: 상속 금지
	(2) 멤버 메소드 앞에 붙일 경우: override 금지
    	(3) 멤버변수 앞에 붙일 경우: 상수화 (변경 금지)
